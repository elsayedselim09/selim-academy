1
الكود الذي تضيفه في <head> لكل صفحة HTML

أضف هذا الكود داخل <head> في index.html و dashboard.html و login.html — بعد سطر meta viewport مباشرةً:

<!-- ═══ PWA — Progressive Web App ═══ -->
<link rel="manifest" href="/selim-academy/manifest.json">
<meta name="theme-color" content="#1A56DB">
<meta name="mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="default">
<meta name="apple-mobile-web-app-title" content="أكاديمية سليم">
<link rel="apple-touch-icon" href="/selim-academy/icons/icon-192x192.png">
<link rel="apple-touch-icon" sizes="152x152" href="/selim-academy/icons/icon-152x152.png">
<link rel="apple-touch-icon" sizes="192x192" href="/selim-academy/icons/icon-192x192.png">
<link rel="icon" type="image/png" sizes="192x192" href="/selim-academy/icons/icon-192x192.png">
<link rel="icon" type="image/png" sizes="512x512" href="/selim-academy/icons/icon-512x512.png">
💡
أسطر apple-mobile-web-app-* ضرورية لـ iOS لأن Safari لا يدعم manifest بشكل كامل.
2
السطر الذي تضيفه قبل </body> في كل صفحة

أضف هذا السطر الوحيد قبل إغلاق </body> في كل صفحة:

<script src="/selim-academy/pwa-register.js" defer></script>
✅
هذا السطر يتولى تسجيل الـ Service Worker تلقائياً + عرض زر "أضف للشاشة الرئيسية" على Android.
3
رفع الملفات على GitHub

ارفع هذه الملفات إلى root مجلد selim-academy في الـ repository:

  • manifest.json
  • sw.js
  • pwa-register.js
  • offline.html
  • مجلد icons/ كاملاً بكل ملفاته الـ 8

ثم عدّل الصفحات الثلاث (index, dashboard, login) بإضافة الكود من الخطوات 1 و 2.

⚠️
مهم: الـ Service Worker يجب أن يكون في /selim-academy/sw.js وليس في أي مجلد فرعي، وإلا لن يتمكن من التحكم في الصفحات.
4
ملاحظات iOS (iPhone/iPad) — مهمة جداً
❌
Safari على iOS لا يدعم beforeinstallprompt، أي زر "تثبيت" تلقائي لن يظهر على iPhone. المستخدم لازم يضيف التطبيق يدوياً.

طريقة الإضافة اليدوية على iOS:

  • افتح الموقع في Safari (مش Chrome)
  • اضغط أيقونة المشاركة (Share) ← المربع اللي فيه سهم للأعلى
  • اختر "Add to Home Screen"
  • اضغط "Add" — التطبيق هيتفتح standalone مباشرةً
💡
يمكنك إضافة بانر توجيهي للمستخدمين على iOS بالكود التالي:
// أضف هذا في pwa-register.js بعد السطر الأول
var isIOS = /iphone|ipad|ipod/i.test(navigator.userAgent);
var isStandaloneIOS = window.navigator.standalone === true;

if (isIOS && !isStandaloneIOS) {
  var iosBanner = document.createElement('div');
  iosBanner.innerHTML =
    '📱 لتثبيت التطبيق: اضغط المشاركة ← إضافة إلى الشاشة الرئيسية' +
    '  ';
  Object.assign(iosBanner.style, {
    position: 'fixed', bottom: '20px', right: '16px', left: '16px',
    background: '#1A56DB', color: '#fff', padding: '14px 16px',
    borderRadius: '12px', fontSize: '.88rem', zIndex: '9999',
    fontFamily: 'Cairo, sans-serif', textAlign: 'center',
    boxShadow: '0 8px 32px rgba(0,0,0,0.2)'
  });
  document.body.appendChild(iosBanner);
}

كيف تتأكد أن الموقع بقى PWA حقيقي

🔍 باستخدام Chrome DevTools (الأسرع):

  • افتح https://elsayedselim09.github.io/selim-academy/ في Chrome
  • اضغط F12 → تبويب Application
  • من القائمة: Service Workers → تأكد إن Status = activated and running
  • من القائمة: Manifest → تأكد من ظهور جميع البيانات والأيقونات
  • من القائمة: Cache Storage → تأكد من وجود selim-academy-v2

🏆 باستخدام Lighthouse (الأشمل):

  • F12 → تبويب Lighthouse
  • اختر فقط: Progressive Web App
  • اضغط Analyze page load
  • يجب أن تحصل على ✅ في: Installable, PWA Optimized

📱 الاختبار الحقيقي على موبايل:

  • افتح الموقع على Chrome Android
  • يجب أن يظهر Banner "أضف للشاشة الرئيسية" تلقائياً
  • بعد الإضافة: افتح التطبيق من الشاشة الرئيسية
  • تأكد من غياب شريط URL بالكامل ✅
  • تأكد من ظهور اسم التطبيق "أكاديمية سليم" في الأعلى
✅
يمكنك أيضاً استخدام https://www.pwabuilder.com — أدخل رابط موقعك وهيعطيك تقرير شامل.
6
مشاكل شائعة في GitHub Pages مع PWA وحلولها

❌ المشكلة 1: Service Worker يظهر خطأ 404

السبب: sw.js مش موجود في مسار /selim-academy/sw.js بالضبط.
الحل: تأكد أن الملف في root مجلد المشروع وليس في مجلد فرعي. تحقق من URL مباشرة: https://elsayedselim09.github.io/selim-academy/sw.js

❌ المشكلة 2: manifest.json يظهر scope error

السبب: start_url أو scope خاطئ.
الحل: تأكد من وجود trailing slash: "start_url": "/selim-academy/" وليس "/selim-academy"

❌ المشكلة 3: التطبيق يفتح بالـ URL Bar (مش Standalone)

السبب: display في manifest.json ليس standalone.
الحل: تأكد من "display": "standalone" في manifest.json. بعد التعديل، احذف التطبيق من الشاشة الرئيسية وأضفه مجدداً.

❌ المشكلة 4: HTTPS Error (نادر على GitHub Pages)

لا مشكلة: GitHub Pages يدعم HTTPS تلقائياً ✅ وهذا شرط أساسي للـ PWA.

❌ المشكلة 5: الكاش القديم يمنع التحديثات

الحل: عند أي تحديث للموقع، غيّر رقم النسخة في sw.js: const CACHE_NAME = 'selim-academy-v3'; (v2 ← v3 إلخ)

❌ المشكلة 6: الأيقونات لا تظهر

الحل: تأكد من رفع مجلد icons/ كاملاً وأن المسارات في manifest.json تبدأ بـ /selim-academy/icons/
✅
